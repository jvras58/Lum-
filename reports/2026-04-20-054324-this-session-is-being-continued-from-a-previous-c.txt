      278  ## 10. Acionar a consolidação de emails manualmente
      279  
      280 -O endpoint `/api/cron/consolidate-emails` é chamado por um sch
          -eduler externo (cron, GitHub Actions, etc.). Para testar em de
          -senvolvimento:                                                
      280 +### Como funciona a autorização (`SECRET_CRON_TOKEN`)         
      281  
      282 +O endpoint `/api/cron/consolidate-emails` é protegido por um t
          +oken Bearer estático. A cada requisição o servidor lê o header
          + `Authorization`, extrai o token e o compara com `SECRET_CRON_
          +TOKEN` (variável server-only do `.env.local`). Se o token esti
          +ver ausente ou incorreto, o endpoint responde `401 Unauthorize
          +d` e não executa nada.                                        
      283 +                                                              
      284 +```                                                           
      285 +Requisição  →  Authorization: Bearer <token>                  
      286 +                       ↓                                      
      287 +               token === SECRET_CRON_TOKEN?                   
      288 +               ├── Sim → executa consolidação → 200 { ok: true
          +, ... }                                                       
      289 +               └── Não → 401 { error: "Unauthorized" }        
      290 +```                                                           
      291 +                                                              
      292 +**Gerar um token seguro:**                                    
      293 +                                                              
      294  ```bash
      295 +# Linux / macOS / Git Bash / WSL                              
      296 +openssl rand -hex 32                                          
      297 +                                                              
      298 +# PowerShell                                                  
      299 +[System.Convert]::ToBase64String((1..32 | ForEach-Object { Get
          +-Random -Maximum 256 }))                                      
      300 +```                                                           
      301 +                                                              
      302 +Copie o valor gerado para `SECRET_CRON_TOKEN` no `.env.local`.
          + Nunca exponha esse token no código-fonte nem em variáveis pre
          +fixadas com `NEXT_PUBLIC_`.                                   
      303 +                                                              
      304 +---                                                           
      305 +                                                              
      306 +### Chamar o endpoint em desenvolvimento                      
      307 +                                                              
      308 +```bash                                                       
      309  curl -X POST http://localhost:3000/api/cron/consolidate-emails
            \
      310    -H "Authorization: Bearer <SECRET_CRON_TOKEN>" \
      311    -H "Content-Type: application/json"
     ...
      326    -d '{"date": "2026-04-20"}'
      327  ```
      328  
      329 +### Agendar com GitHub Actions (exemplo)                      
      330 +                                                              
      331 +Crie `.github/workflows/consolidate-emails.yml` no repositório
          +:                                                             
      332 +                                                              
      333 +```yaml                                                       
      334 +name: Consolidar emails diários                               
      335 +                                                              
      336 +on:                                                           
      337 +  schedule:                                                   
      338 +    - cron: "0 22 * * *"   # todo dia às 22 h UTC             
      339 +  workflow_dispatch:        # permite disparo manual pelo pain
          +el do GitHub                                                  
      340 +                                                              
      341 +jobs:                                                         
      342 +  consolidate:                                                
      343 +    runs-on: ubuntu-latest                                    
      344 +    steps:                                                    
      345 +      - name: Chamar endpoint de consolidação                 
      346 +        run: |                                                
      347 +          curl -X POST ${{ secrets.APP_URL }}/api/cron/consoli
          +date-emails \                                                 
      348 +            -H "Authorization: Bearer ${{ secrets.SECRET_CRON_
          +TOKEN }}" \                                                   
      349 +            -H "Content-Type: application/json" \             
      350 +            --fail                                            
      351 +```                                                           
      352 +                                                              
      353 +Configure dois **Secrets** no repositório (Settings → Secrets 
          +→ Actions):                                                   
      354 +                                                              
      355 +| Secret | Valor |                                            
      356 +|--------|-------|                                            
      357 +| `APP_URL` | URL pública da aplicação, ex.: `https://lum.exem
          +plo.com` |                                                    
      358 +| `SECRET_CRON_TOKEN` | O mesmo valor definido no `.env.local`
          + / variáveis de ambiente do host |
